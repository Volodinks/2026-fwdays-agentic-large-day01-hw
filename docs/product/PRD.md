# PRD: Excalidraw (reverse-engineered)

## 1) Огляд продукту

Excalidraw - це веб-редактор для швидкого візуального мислення у hand-drawn стилі. Продукт дозволяє створювати діаграми та схеми, редагувати їх у браузері, зберігати/експортувати результат, ділитися артефактами та працювати в колаборативному режимі.

Цей PRD є reverse-engineered документом, сформованим на базі наявного коду та технічної документації монорепозиторію.

## 2) Мета продукту

- Дати користувачам простий і швидкий інструмент для створення візуальних артефактів без складного онбордингу.
- Забезпечити стабільний редактор, який працює локально в браузері, підтримує імпорт/експорт та колаборацію.
- Надати інтеграторам publishable React-компонент `<Excalidraw />` та імперативний API для вбудовування редактора у сторонні застосунки.

## 3) Цільова аудиторія

### Primary users

- Продуктові команди (PM, дизайнери, розробники) для брейнштормів, схем флоу та обговорення рішень.
- Інженерні команди для технічних діаграм, архітектурних ескізів і документації.
- Освітні/контент-команди для швидких пояснювальних візуалізацій.

### Secondary users

- Інтегратори та платформні команди, яким потрібен embedded whiteboard через `@excalidraw/excalidraw`.
- Open-source контриб'ютори, які розвивають функції редактора в монорепо.

## 4) Проблеми, які вирішує продукт

- Складні діаграмні інструменти мають високий поріг входу для швидких нотаток.
- Командам потрібен єдиний формат для створення, редагування та обміну діаграмами.
- Для embedded-сценаріїв потрібен стабільний API, щоб керувати сценою програмно (`updateScene`, `onChange`, library operations).

## 5) Ключові користувацькі сценарії

- Як користувач, я хочу швидко намалювати діаграму на canvas і зберегти результат.
- Як користувач, я хочу імпортувати існуючу сцену/файли та експортувати готовий артефакт.
- Як учасник команди, я хочу редагувати схему разом з іншими в live-режимі.
- Як інтегратор, я хочу вбудувати редактор у свій React-застосунок і керувати сценою через API.

## 6) Функціональні вимоги (MVP/current product)

### Редактор і взаємодія

- Полотно для створення та редагування елементів сцени (`ExcalidrawElement`).
- Базові інструменти: selection, геометричні фігури, стрілки, текст, image, frame, hand, eraser, laser.
- Керування станом редактора (`AppState`): активний інструмент, selection, viewport, модальні стани, тощо.
- Дії/команди через action system (`ActionManager`, keyboard shortcuts, UI actions).

### Дані сцени та стійкість

- Модель `Scene` як авторитетний контейнер елементів (включно з versioning/pорядком).
- Відновлення сцени та app state під час ініціалізації.
- Збереження змін через `onChange` у хост-додатку.
- Підтримка бібліотеки reusable-фігур (`Library`, `LibraryItem`).

### Колаборація

- Підключення до кімнати та live-синхронізація елементів сцени.
- Обробка collaborator presence (курсори, стани учасників, список користувачів).
- Reconciling remote updates з локальною сценою.

### Інтеграційний API

- Публічний компонент `<Excalidraw />` для React.
- Імперативний API (`ExcalidrawImperativeAPI`) для програмного доступу до сцени, app state, файлів і бібліотеки.
- Приклади інтеграції в `examples/*`.

## 7) Нефункціональні вимоги

- Продуктивність: плавна взаємодія на canvas, окремі рендер-канваси для статичного, інтерактивного та preview шарів.
- Надійність: коректне збереження стану, undo/redo через store/history pipeline.
- Розширюваність: модульна архітектура монорепо (`packages/*` + `excalidraw-app`).
- Якість коду: typecheck, lint, тести у CI.
- Портативність: робота у браузері, dev через Vite, production build для статичного хостингу.

## 8) Технічні обмеження та рамки

- Monorepo на Yarn workspaces (`excalidraw-app`, `packages/*`, `examples/*`).
- Мінімальна версія Node.js: `>=18`.
- Основний стек: TypeScript, React, Vite, Vitest.
- Істотні продуктові інтеграції (Firebase, Sentry, socket.io) налаштовуються через app-layer та env-конфігурацію.
- Архітектурне розділення:
  - ядро редактора і публічний API - `packages/excalidraw`
  - елементна/сценова модель - `packages/element`
  - продуктова обгортка та оркестрація інтеграцій - `excalidraw-app`

## 9) Scope / Non-goals

### In scope

- Редагування сцени в браузері.
- Імпорт/експорт, базова персистентність, колаборація.
- Публічний API для вбудовування редактора.

### Out of scope

- Детальний roadmap майбутніх фіч (поза межами reverse-engineered PRD).
- Специфікація backend-реалізації поза тим, що безпосередньо спостерігається в репозиторії.

## 10) Критерії успіху

- Користувач може створити, відредагувати та зберегти/експортувати сцену без помилок.
- Колабораційна сесія коректно синхронізує ключові зміни сцени між учасниками.
- Інтегратор може вбудувати `<Excalidraw />` і виконати базовий сценарій керування через API.
- Репозиторій проходить базові перевірки якості (typecheck/lint/tests) у CI.

## 11) Відкриті питання

- Формалізувати продуктові метрики (активація, retention, collaborative session success rate).
- Винести узгоджений перелік env-параметрів для інтеграцій у публічну документацію.
- Уточнити версійовані гарантії сумісності для публічного API (`@excalidraw/excalidraw`).
