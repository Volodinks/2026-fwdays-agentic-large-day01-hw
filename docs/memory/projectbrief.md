# Project brief

## Суть проєкту

Це **Yarn workspaces монорепозиторій** (root `"name": "excalidraw-monorepo"`) з екосистемою **Excalidraw**:

- **Продуктовий веб-додаток** `excalidraw-app`: повноцінний редактор на полотні для створення діаграм/схем у “hand‑drawn” стилі.
- **Бібліотека** `@excalidraw/excalidraw` (`packages/excalidraw`): React-компонент `<Excalidraw />` і публічний API для вбудовування редактора в інші застосунки.

## Місія

Надати простий, швидкий і надійний інструмент для візуального мислення та комунікації: від швидких ескізів до діаграм, які легко **створювати, редагувати, ділитися та вбудовувати**.

## Головна мета

- **Для кінцевих користувачів**: зручний браузерний редактор із ключовими можливостями (малювання/редагування, бібліотека фігур, імпорт/експорт, теми, локалізації, PWA).
- **Для інтеграторів**: стабільний API + типи + стилі для `<Excalidraw />`, а також референсні приклади інтеграції (`examples/*`).

## Цілі (у межах цього репозиторію)

- Підтримувати **узгоджений розвиток** додатку та бібліотеки в одному кодбейзі.
- Мати **відтворювану збірку та CI** для app і пакетів.
- Тримати чіткі межі: “ядро редактора” в `packages/*`, продуктові інтеграції — в `excalidraw-app`.

## Обсяг (scope)

### Ключові частини репозиторію

| Частина | Призначення |
|--------|-------------|
| `excalidraw-app/` | Хост-додаток (Vite + React), інтеграції та деплой (PWA, Sentry, колаб, Firebase тощо). |
| `packages/excalidraw/` | Основна publishable бібліотека з UI та логікою редактора. |
| `packages/common`, `packages/math`, `packages/element`, `packages/utils` | Доменні/допоміжні пакети (модель елементів, геометрія, утиліти). |
| `examples/*` | Приклади інтеграції (Next.js, “script in browser”). |
| `scripts/` | Скрипти збірки/релізу (зокрема esbuild пайплайн для пакетів). |

### Продуктовий шар `excalidraw-app`

`excalidraw-app` є обгорткою навколо `@excalidraw/excalidraw` і додає продуктові інтеграції (наприклад: колаборація, Firebase, app-level стан, Sentry).

## Критерії успіху (high-level)

- **App** запускається локально і збирається в продакшн без помилок.
- **Пакети** збираються та можуть бути використані як бібліотека у прикладах.
- Базові перевірки якості (typecheck/lint/tests) відпрацьовують у CI.
